- Tool Calling (Function Calling)
    
    ```jsx
    Tool : LLM이 외부 세계와 상호작용하기 위해 사용하는 인터페이스
           특정 작업을 수행하는 함수(날씨 정보 검색, 데이터베이스 조회) 형태로 제공
    
    Tool Calling : LLM이 외부 API를 호출하거나, 특정 함수를 실행하거나, 다른 소프트웨어와 상호작용하는 것을 말함.
    ```
    
    ```jsx
    1. 도구 정의 : Agent가 사용할 수 있는 도구 정의
    2. 도구 설명 : LLM에게 각 도구의 기능, 입력/출력 형식 등을 설명
    3. LLM 추론 : LLM은 사용자 요청과 도구 설명 기반으로 어떤 도구를 호출할지, 어떤 파라미터를 사용할지 결정
    4. 도구 실행 : LLM의 요청에 따라 실제 도구를 실행하고, 그 결과를 LLM에게 반환함
    5. 결과 종합 : LLM은 도구 실행 결과를 종합하여 사용자에게 최종 응답 제공
    ```
    
    [Agent의 기반이 되는 LLM Tool Calling: ① 개념 알기](https://uracle.blog/2025/03/28/agent%EC%9D%98-%EA%B8%B0%EB%B0%98%EC%9D%B4-%EB%90%98%EB%8A%94-llm-tool-calling/)

- Agent Loop 구조 (LLM → Tool → LLM)
    
    ```jsx
    1. 입력 프롬프트 준비
    	 토큰 기반으로 정수 시퀀스로 변환되어 모델에게 전달됨.
    	 
    2. 모델 추론
    	 도구 호출
    	 
    3. 도구 실행
    ```
